# Подробный разбор `script.js`

Этот файл нужен как расшифровка к `chart/script.js`.
Я специально не повторяю базовые вещи вроде того, что `const` создает неизменяемую ссылку, а `let` - изменяемую.
Вместо этого здесь объясняется, какая именно роль у каждой строки и почему она нужна.

## 1. Общая структура файла

Файл состоит из 5 больших частей:

1. Подготовка элементов интерфейса и глобального состояния.
2. Коллатц-логика и вспомогательные функции форматирования.
3. Построение основного графика.
4. Построение круговой диаграммы.
5. Подсчет статистики и обработчики событий.

```js

```
Что делает:
- Находит поле ввода с `id="input"`.
- Сохраняет ссылку на него, чтобы потом быстро читать введенное число и управлять фокусом.
- Это основной источник пользовательского ввода для расчетов.

### Строка
```js

```
Что делает:

- Эта кнопка используется, чтобы начать расчет последовательности Collatz.
- На нее навешан обработчик клика ниже в файле.

### Строка

Что делает:

### Строки
```js

```
Что делает:
- `useLogScale` хранит режим отображения: обычная шкала или логарифмическая.
- `currentSequence` хранит последнюю рассчитанную последовательность, чтобы ее можно было перерисовать без повторного ввода.
- Это состояние нужно, чтобы переключение масштаба не требовало повторного запуска алгоритма.

## 3. Переключение шкалы

### Функция
```js
function setScale(mode) {
    useLogScale = (mode === 'log');
    document.getElementById('btnLinear').classList.toggle('active', !useLogScale);
    document.getElementById('btnLog').classList.toggle('active',  useLogScale);
    if (currentSequence.length) renderGraph(currentSequence, useLogScale);
}
```

Разбор по строкам:

### `useLogScale = (mode === 'log');`
- Проверяет, был ли передан режим `log`.
- Если да, включает логарифмический режим.
- Если нет, возвращает линейный режим.
- Это делает функцию универсальной для двух кнопок или двух вариантов выбора.

### `document.getElementById('btnLinear').classList.toggle('active', !useLogScale);`
- Находит кнопку линейной шкалы.
- Добавляет или убирает класс `active`.
- Если логарифмический режим выключен, кнопка линейной шкалы подсвечивается как активная.

### `document.getElementById('btnLog').classList.toggle('active',  useLogScale);`
- То же самое, но для кнопки логарифмического режима.
- Когда `useLogScale = true`, именно эта кнопка выглядит активной.

### `if (currentSequence.length) renderGraph(currentSequence, useLogScale);`
- Проверяет, есть ли уже рассчитанная последовательность.
- Если она существует, график перерисовывается в новом масштабе сразу.
- Это важно: пользователь видит смену масштаба без повторного ввода числа.

## 5. Построение последовательности Collatz

### Функция
```js
function collatzBigInt(n) {
    const seq = [];
    if (n === 0n) return seq;
    while (true) {
        seq.push(n);
        if (n === 1n) break;
        n = (n % 2n === 0n) ? n / 2n : n * 3n + 1n;
        if (seq.length > 50000) break;
    }
    return seq;
}
```

### `const seq = [];`
- Создает массив для всей последовательности.
- В него будут добавляться все промежуточные значения от стартового числа до `1`.

### `if (n === 0n) return seq;`
- Если пользователь ввел `0`, функция сразу возвращает пустой массив.
- Для Collatz-цепочки это защитная проверка, потому что стандартная последовательность обычно рассматривается для положительных чисел.

### `while (true) {`
- Запускает цикл без заранее известного числа итераций.
- Он завершится только тогда, когда достигнется `1` или сработает ограничение длины.

### `seq.push(n);`
- Добавляет текущее значение в историю последовательности.
- Именно так строится полный путь числа.

### `if (n === 1n) break;`
- Если достигнуто `1`, цепочка завершена.
- В классической задаче Collatz это конечная точка.

### `n = (n % 2n === 0n) ? n / 2n : n * 3n + 1n;`
- Это сама формула Collatz.
- Если число четное, оно делится на `2`.
- Если число нечетное, применяется правило `3n + 1`.
- Запись через тернарный оператор делает выбор между двумя ветками компактным.

### `if (seq.length > 50000) break;`
- Защита от слишком длинных вычислений.
- Если последовательность стала слишком большой, цикл принудительно останавливается.
- Это предотвращает зависание интерфейса на экстремальных значениях.

### `return seq;`
- Возвращает готовую последовательность.
- Далее она используется для графика, статистики и подсказок.

## 6. Логарифм для очень больших `BigInt`

### Функция
```js
function bigIntLog10(value) {
    const s = value.toString();
    const k = 15;
    if (s.length <= k) return Math.log10(Number(s));
    const lead = Number(s.slice(0, k));
    return Math.log10(lead) + (s.length - k);
}
```

### `const s = value.toString();`
- Переводит большое число в строку.
- Это нужно, чтобы работать не с самим `BigInt`, а с его десятичной записью.

### `const k = 15;`
- Определяет, сколько первых цифр берется для приближенного расчета.
- 15 цифр - это компромисс между точностью и удобством.

### `if (s.length <= k) return Math.log10(Number(s));`
- Если число короткое, можно без проблем преобразовать его в обычный `Number`.
- В таком случае логарифм считается напрямую.

### `const lead = Number(s.slice(0, k));`
- Берет первые 15 значащих цифр.
- По ним строится приближение логарифма для огромного числа.

### `return Math.log10(lead) + (s.length - k);`
- Суммирует логарифм от ведущих цифр и поправку на длину числа.
- Это позволяет оценивать `log10` даже для астрономически больших значений.

## 7. Сокращение длинных чисел для подсказок

### Функция
```js
function toolTipShorten(value) {
    const s = value.toString();
    if (s.length <= 15) return s;
    return `${s.slice(0, 6)}...${s.slice(-6)}`;
}
```

### Что делает:
- Превращает большое число в короткую читаемую строку.
- Если число короткое, возвращает его полностью.
- Если число длинное, оставляет первые 6 и последние 6 цифр.
- Это особенно полезно в tooltip и таблицах, где полное число было бы слишком длинным.

## 8. Форматирование оси X

### Функция
```js
function formatAxisLabel(v) {
    if (v <= 0) return '';
    if (v < 1e6) return Math.round(v).toLocaleString('en-US');
    const exp = Math.floor(Math.log10(v));
    const mant = (v / Math.pow(10, exp)).toFixed(1).replace(/\.0$/, '');
    return `${mant}e+${exp}`;
}
```

### `if (v <= 0) return '';`
- Не показывает отрицательные и нулевые подписи.
- Для оси значений это упрощает внешний вид.

### `if (v < 1e6) return Math.round(v).toLocaleString('en-US');`
- Для небольших чисел выводит обычное округленное значение.
- `toLocaleString('en-US')` добавляет разделители тысяч.

### `const exp = Math.floor(Math.log10(v));`
- Находит порядок числа.
- Например, для `1230000` это будет `6`.

### `const mant = (v / Math.pow(10, exp)).toFixed(1).replace(/\.0$/, '');`
- Вычисляет мантиссу - число в виде от `1` до `10`.
- Если после округления получается `.0`, эта часть убирается.

### `return \`${mant}e+${exp}\`;`
- Показывает число в научной записи.
- Это удобно для очень больших значений оси.

## 9. Форматирование оси Y в лог-режиме

### Функция
```js
function formatLogAxisLabel(v) {
    if (v < 0) return '';
    if (v === 0) return '1';
    const actual = Math.pow(10, v);
    if (actual < 1e4) return Math.round(actual).toLocaleString('en-US');
    const exp = Math.round(v);
    if (Math.abs(v - exp) < 0.02) return `10^${exp}`;
    return `10^${v.toFixed(1)}`;
}
```

### Что делает:
- Преобразует значение логарифмической оси обратно в понятную человеку запись.
- Если на оси стоит `0`, это соответствует `10^0 = 1`.
- Если реальное значение еще небольшое, показывается обычное число.
- Для больших значений выводится вид `10^n`, чтобы не перегружать подписи.

## 10. Умный зум

### Функция
```js
function calculateSmartZoom(sequence, isLog) {
    if (isLog || sequence.length <= 50) return { start: 0, end: 100 };
    let max = 0n;
    for (const x of sequence) if (x > max) max = x;
    const threshold = max / 100n;
    let peakIndex = sequence.indexOf(max);
    if (peakIndex === -1) peakIndex = 0;
    for (let i = peakIndex; i < sequence.length; i++) {
        if (sequence[i] < threshold) {
            const end = Math.min(100, Math.round((i + 30) / sequence.length * 100));
            return { start: 0, end };
        }
    }
    return { start: 0, end: 100 };
}
```

### `if (isLog || sequence.length <= 50) return { start: 0, end: 100 };`
- В лог-режиме автоматический зум не нужен.
- Для коротких последовательностей тоже нет смысла обрезать диапазон.

### Поиск максимума
- Цикл ищет наибольшее значение в последовательности.
- Это пик цепочки.

### `const threshold = max / 100n;`
- Задает порог в 1% от максимума.
- После пика график часто очень быстро падает, и этот порог помогает найти момент, после которого линия уже не интересна.

### `let peakIndex = sequence.indexOf(max);`
- Находит индекс максимального значения.
- С него начинается поиск момента, когда значения снова падают.

### Цикл после пика
- Идет от максимума к концу последовательности.
- Как только число опускается ниже порога, вычисляется конечный предел зума.

### `return { start: 0, end };`
- Возвращает диапазон от начала до рассчитанного конца.
- Это делает график более компактным и полезным по умолчанию.

```js
const yData = isLog
```

- Маленькие числа переводятся в обычный `Number`.
- Слишком большие числа приближаются через первые 15 цифр и степень десяти.

### `const GLOW  = '#e24bde';`
- Хранят основные цвета линии и свечения.

### Сборка текста шагов
```js
const allSteps = sequence.map((val, idx) => {
```

Что происходит:

- `op` показывает, какая операция была применена к текущему числу.
- `join('\n')` превращает массив строк в один текстовый блок.

## 14. Tooltip

```js
tooltip: {
    trigger: 'axis',
    ...
    formatter(params) {
        const idx = params[0].dataIndex;
        const val = sequence[idx];
        const isEven = val % 2n === 0n;
        const realVal = toolTipShorten(val);
        const logLine = isLog
            ? `<div style="color:#a78bfa;font-size:11px">log₁₀ ≈ ${params[0].value.toFixed(4)}</div>`
            : '';
        return `<div style="font-weight:600;margin-bottom:4px">Step #${idx}</div>
                <div style="font-family:monospace">${realVal}</div>
                ${logLine}
                <div style="color:${isEven ? '#34d399' : '#f87171'};margin-top:2px">
                    ${isEven ? '÷ 2' : '× 3 + 1'}
                </div>`;
    }
}
```

Что делает:
- Показывает всплывающую подсказку при наведении на точку.
- `params[0]` содержит данные текущего шага.
- `dataIndex` нужен, чтобы взять исходное значение из `sequence`.
- `isEven` определяет, какая операция применялась к числу.
- `realVal` выводит короткую читаемую форму числа.
- В лог-режиме добавляется строка с приближенным `log10`.
- Цвет строки меняется в зависимости от четности.

## 22. Обновление статистики

### Функция
```js
function updateInterface(sequence) {
    if (!sequence.length) return;
    ...
}
```

Что делает:
- Заполняет все карточки и текстовые поля на странице.
- Работает только если последовательность не пустая.
- Берет вычисленные значения и превращает их в интерфейсные цифры.

### Максимум, индекс пика и рост
```js
const maxVal    = sequence.reduce((a, b) => a > b ? a : b, 0n);
const peakIndex = sequence.indexOf(maxVal);
const totalSteps = sequence.length - 1;
const growth    = maxVal / sequence[0];
```

- `maxVal` - наибольшее число в цепочке.
- `peakIndex` - позиция этого значения.
- `totalSteps` - число переходов между элементами, а не просто длина массива.
- `growth` - во сколько раз максимум больше стартового числа.

### Серии подъема и падения
```js
let maxUp = 0, maxDown = 0, curUp = 0, curDown = 0;
for (let i = 1; i < sequence.length; i++) {
    if (sequence[i] > sequence[i - 1]) { curUp++;   curDown = 0; maxUp   = Math.max(maxUp,   curUp);   }
    else                                { curDown++; curUp   = 0; maxDown = Math.max(maxDown, curDown); }
}
```

Что делает:
- Считает самую длинную подряд идущую серию роста.
- Считает самую длинную подряд идущую серию падения.
- На каждом шаге одна серия сбрасывается, а другая увеличивается.

### Среднее и медиана
```js
const sum  = sequence.reduce((a, v) => a + v, 0n);
const avg  = sum / BigInt(sequence.length);
const sorted = [...sequence].sort((a, b) => a < b ? -1 : 1);
const mid  = Math.floor(sorted.length / 2);
const median = sorted.length % 2 !== 0 ? sorted[mid] : (sorted[mid - 1] + sorted[mid]) / 2n;
```

Что делает:
- `sum` собирает сумму всех элементов.
- `avg` считает среднее арифметическое.
- `sorted` сортирует копию последовательности по возрастанию.
- `median` берет центральное значение или среднее двух центральных, если элементов четное число.

### Подсчет четных и нечетных
```js
let even = 0, odd = 0;
sequence.forEach(v => { if (v % 2n === 0n) even++; else odd++; });
```
- Проходит по всей последовательности.
- Считает, сколько значений четные, а сколько нечетные.
- Эти данные идут в круговую диаграмму и проценты.

### Проценты
```js
const evenPct = ((even / sequence.length) * 100).toFixed(0);
const oddPct  = ((odd  / sequence.length) * 100).toFixed(0);
```
- Переводит количество в проценты.
- `toFixed(0)` оставляет только целые проценты, чтобы интерфейс был проще.

### Функция `fmt`
```js
const fmt = (n) => {
    const s = n.toString();
    if (s.length <= 15) return BigInt(n).toLocaleString('en-US');
    return `${s.slice(0, 6)}…${s.slice(-6)}`;
};
```

Что делает:
- Унифицирует формат показа больших чисел в интерфейсе.
- Короткие числа показывает полностью.
- Длинные - в сокращенном виде.

### Обновление карточек
```js
document.getElementById('card-steps').textContent  = totalSteps;
document.getElementById('card-peak').textContent   = fmt(maxVal);
document.getElementById('card-growth').textContent = `${growth}x`;
```
- Заполняет верхние сводные карточки.
- Пользователь видит число шагов, максимум и рост последовательности.

### Обновление подробных статистик
- `stat-start` показывает стартовое число.
- `stat-steps` показывает длину цепочки.
- `stat-peak` показывает максимум.
- `stat-peak-step` показывает, на каком шаге был пик.
- `stat-growth` показывает рост.
- `stat-time-peak` дублирует момент пика в более читаемом виде.
- `stat-streak-up` и `stat-streak-down` показывают самые длинные серии роста и падения.
- `stat-avg` и `stat-median` показывают среднее и медиану.

### Обновление процентов и центра круга
```js

document.getElementById('pct-odd').textContent      = `${oddPct}%`;
```

- Выводит долю четных и нечетных чисел в текстовом виде.

### Обновление Chart.js

